# Security Policy

## Driver Safety

This app must only install WinUSB on explicit supported control interfaces.
For the first validated profile, the only allowed target is:

```text
USB\VID_0471&PID_0111&MI_03
```

The app must not install WinUSB on:

```text
USB\VID_0471&PID_0111&MI_00
```

`MI_00` is the Windows USB Audio interface and must remain on the normal
Microsoft audio driver.

## Reporting Issues

When reporting a problem, attach diagnostics generated by the app and include:

- Philips model number.
- Windows version.
- Whether the stereo is in USB PC Link mode.
- Device Manager hardware IDs for the Philips device.
- Whether audio works but is quiet, absent, or distorted.

Do not attach private music files or unrelated system logs.

## Release Signing

Community builds may use the bundled libwdi-generated test driver package.
A production release should use:

- A code-signed application executable.
- A production-signed driver package.
- Published checksums for release archives.
